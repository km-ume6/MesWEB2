# OCR 誤認識改善 - 実装内容

## 改善された問題

以下の OCR 誤認識が修正されました：

1. **"Pes:" → "Pcs:"** - よくある誤認識パターン
2. **"Pcs,:" → "Pcs:"** - コロンとカンマの混同
3. **"0%" → "01"** - パーセント記号と数字の誤認識
4. **"gl" → "01"** - 小文字の g と l が 01 に誤認識される
5. **"O" → "0"** - 文字の O と数字の 0 の混同

## ラベル検出問題の修正（最新）

### 問題
- ラベル2と3が正しく検出されず、"-" と "o" のみが抽出される
- ROI（関心領域）検出が失敗している

### 解決策

#### 1. フォールバック戦略の追加
キーワード（"Date" と "compatible"）が検出されない場合の対応：
- **画像の均等分割**: 画像を垂直方向に3分割してROIを作成
- **参照ベースROI作成**: Date または compatible のいずれかが見つかった場合、その位置を基準にROIを作成

#### 2. キーワード検出の柔軟化
- **部分一致**: "Date" の代わりに "Dat" や "ate" でも検出
- **スペルミス許容**: Levenshtein距離が2以下なら許容（例: "Dute" → "Date"）
- **近似マッチング**: より多様な表記を検出可能に

#### 3. ペアリングロジックの改善
- **垂直距離の制約を緩和**: 600px → 800px に拡大
- **上方向の許容**: Compatible が Date より上にあっても100px以内なら許容
- **詳細なデバッグログ**: 各ペアリングの判定理由をログ出力

#### 4. 詳細なログ出力
各検出ステップで以下を記録：
- 検出された Date と compatible の座標
- ペアリングの候補と判定理由
- フォールバック戦略の発動理由

### 2. 文字修正処理の改善

#### 新規追加: `CorrectCommonMisrecognitions()` メソッド
すべての誤認識修正を統合的に処理:
- "Pes:" → "Pcs:" の修正
- "Pcs,:" → "Pcs:" の修正
- 数字パターンの修正
- 0/O の混同修正

#### 新規追加: `CorrectNumberPatterns()` メソッド
数字に関連する誤認識を包括的に修正:

```csharp
// "gl" → "01"
text = Regex.Replace(text, @"\bgl\b", "01", RegexOptions.IgnoreCase);

// "0%" → "01" (Pcs: の後)
text = Regex.Replace(text, @"(Pcs:\s*)0%", "$1 01");
text = Regex.Replace(text, @"(Pcs:\s*)O%", "$1 01");

// "I" → "1" (数字コンテキスト)
text = Regex.Replace(text, @"(\d)I(\s|$|~)", "$1 1$2");

// "l" (小文字L) → "1"
text = Regex.Replace(text, @"(\d)l(\s|$|~)", "$1 1$2");

// "S" → "5"
text = Regex.Replace(text, @"(^|\s|~|Pcs:\s*)S(\d)", "$1 5$2");

// "B" → "8"
text = Regex.Replace(text, @"(^|\s|~)B(\d)", "$1 8$2");
```

## テスト結果の確認方法

アプリケーションを実行して画像をアップロードすると：
1. デバッグ画像が `%TEMP%\ocr_debug_<timestamp>\` に保存される
2. ログに各処理ステップの結果が記録される
   - キーワードの検出位置
   - ペアリングの判定理由
   - フォールバック戦略の発動有無
3. 抽出されたテキストで以下が修正されているか確認:
   - すべてのラベル（3つ）が検出される
   - "Pcs:" が正しく認識される
   - "01" などの数字が正しく認識される
   - 範囲表記が正しく解析される
